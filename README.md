# Sensitive Data Detection & Compliance Assistant

## Overview

The Sensitive Data Detection & Compliance Assistant is an AI-powered application that analyzes uploaded documents to identify sensitive information, assess compliance risks, and answer user questions using Retrieval-Augmented Generation (RAG).

The application combines deterministic pattern matching using Python Regex with Large Language Models (LLMs) to generate professional compliance reports and provide intelligent document-based question answering.



# Features

## Document Upload

Supports multiple document formats:

- PDF
- TXT
- CSV



## Sensitive Data Detection

The application automatically detects confidential information including:

- Aadhaar Numbers
- PAN Numbers
- Email Addresses
- Phone Numbers
- Credit Card Numbers
- API Keys
- Passwords
- Employee IDs
- Bank Account Details

Detection is performed using Python Regular Expressions (Regex) for fast and reliable extraction.



## Risk Classification

The detected information is analyzed and classified into:

- Low Risk
- Medium Risk
- High Risk

Risk classification is generated by the LLM based on predefined security rules and the detected sensitive information.

---

## AI Generated Compliance Report

The assistant generates:

- Executive Summary
- Risk Assessment
- Compliance Observations
- Security Risks
- Suggested Remediation Steps



## Natural Language Question Answering

Users can ask questions such as:

- What sensitive data exists in the document?
- How many email addresses are present?
- Summarize this document.
- What compliance risks are identified?

The application uses Retrieval-Augmented Generation (RAG) to answer questions based only on the uploaded document.

# AI/ML Approach

This project follows a hybrid AI architecture.

### Step 1

The uploaded document is converted into plain text.

### Step 2

Python Regex extracts structured sensitive information such as:

- PAN
- Aadhaar
- Emails
- Phone Numbers
- Credit Cards
- API Keys

### Step 3

The extracted results are provided to the Groq LLM, which generates:

- Risk Classification
- Compliance Summary
- Security Observations
- Recommended Actions

### Step 4

The complete document is split into semantic chunks using LangChain.

### Step 5

Sentence embeddings are generated using HuggingFace Sentence Transformers.

### Step 6

Embeddings are stored inside a FAISS vector database.

### Step 7

When the user asks a question, the most relevant chunks are retrieved from FAISS and provided to the Groq LLM for accurate, context-aware responses
